# План сборки в Cursor (по фазам)

Каждая фаза = что сказать агенту + что должно получиться (acceptance). Идём по порядку, не перепрыгивая. Для каждой фазы давай Cursor доступ к нужным docs (`@ARCHITECTURE.md` и т.д.).

## Фаза 0. Каркас
**Cursor:** «Создай проект на Astro + TypeScript. Заведи структуру: `src/`, `src/components/`, `src/layouts/`, `src/pages/`, `src/lib/` (адаптеры, поиск), `data/`, `public/models/`, `scripts/` (sync, embed, enrich, fetch-model). Подключи дизайн-токены из `DESIGN-SYSTEM.md` в глобальный CSS (`:root`), Onest + Golos Text. Создай `site.config.ts` (название, токены палитры, контакты, telegram chat id, источник каталога).»
**Готово:** проект запускается (`npm run dev`), токены и шрифты применяются.

## Фаза 1. Слой данных
**Cursor:** «По `DATA-MODEL.md` реализуй тип `Product`, интерфейс `SourceAdapter` и адаптеры: YML, CSV, Google Sheets — с capability-флагами. Добавь нормализацию атрибутов (словарь синонимов). Напиши `scripts/sync` → пишет `data/catalog.json`. Источник берётся из `site.config.ts`.»
**Готово:** `npm run sync` на тестовом YML/таблице даёт валидный `catalog.json`.

## Фаза 2. Витрина (статические страницы)
**Cursor:** «Сгенерируй из `catalog.json`: главную, каталог с фильтрами (авто из `attributes`) и сортировкой, карточку товара (PDP), корзину. Свёрстай по вайрфреймам и компонентам из `DESIGN-SYSTEM.md`. Учитывай фиче-тоглы: нет остатка → «уточняйте наличие», нет вариантов → без переключателя, и т.д. Адаптив + quality floor (фокус, alt, контраст, reduced-motion).»
**Готово:** по каталогу генерятся все страницы, фильтры/сортировка работают, мобильная версия ок.

## Фаза 3. Поиск по фото и тексту (фишка)
**Cursor:** «По `PHOTO-SEARCH.md`: `scripts/embed` считает эмбеддинги картинок (CLIP) → `data/embeddings.json` (L2-норма). `scripts/fetch-model` кладёт квантованный CLIP в `public/models/`. В браузере — компонент поиска: текстовый запрос в строке (лёгкий text-энкодер) и dropzone «поиск по фото» (ленивая загрузка image-энкодера, инференс в браузере, фото не уходит на сервер). Ранжирование dot-product по `embeddings.json`, топ-K → грид карточек. Блок «похожие товары» в PDP на тех же эмбеддингах.»
**Готово:** загрузка фото и текстовый запрос выдают релевантные товары; модель грузится лениво и кешируется; «похожие» работают.

## Фаза 4. Тексты и SEO (GigaChat, build-time)
**Cursor:** «По `GIGACHAT.md`: `scripts/enrich` вызывает GigaChat (ключ из `.env`, build-only) → заполняет `descriptionRich` и `seo` в `catalog.json`, с кешем по хэшу входа. Карточки и `<head>` используют эти поля, с фолбэком на сырое описание. Добавь sitemap, og-теги, schema.org Product.»
**Готово:** у товаров полированные тексты + корректные мета/og/schema; повторный `enrich` не жжёт токены на неизменном.

## Фаза 5. Заказы
**Cursor:** «По `ARCHITECTURE.md §5`: форма заказа в корзине отправляет `POST {tenant, order}` на общий релай. Реализуй сам релай как отдельную Yandex Cloud Function (scale-to-zero): принимает заказ, по `tenant` шлёт email + сообщение в Telegram. Покажи понятные состояния: отправка / «Заказ отправлен» / ошибка с подсказкой. Заложи fallback deep-link (Telegram/wa.me/mailto).»
**Готово:** тестовый заказ доходит в email и Telegram; UI-состояния корректны.

## Фаза 6. Финализация и per-client сборка
**Cursor:** «Прогон quality floor (Lighthouse: перф/доступность/SEO). Документируй процесс заведения клиента: заполнить `site.config.ts`, положить веса в `public/models/`, `sync → embed → enrich → build → deploy`. Проверь, что смена `--accent` перекрашивает весь сайт без правок кода.»
**Готово:** чек-лист «новый клиент за N шагов»; перекраска тенанта = только токены.

---

### Порядок и дисциплина
- Стройся снизу вверх: данные → витрина → поиск → тексты → заказы. Не начинать фазу, пока предыдущая не «зелёная».
- Всё новое уважает canonical-формат и capability-флаги (см. `DATA-MODEL.md`).
- Отложенное (AR/3D, прямой API, админка, конфигуратор) — НЕ трогаем в MVP, но не ломаем под него интерфейсы.
