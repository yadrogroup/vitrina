# GigaChat: тексты карточек и SEO

## Роль и размещение

GigaChat вызывается **только на сборке** (`npm run enrich`), из твоего пайплайна, **одним твоим платным ключом**. В готовом статическом сайте его нет — там лежит готовый текст. Клиенту аккаунт/ключ не нужен.

- Ключ — **ИП/юрлицо** (Freemium физлица — для прототипов; коммерческая генерация клиентам = платный тариф). Токены копеечные; на каталог одного мебельщика уходит немного.
- Ключ в `.env` (`GIGACHAT_API_KEY`), **build-only**, в бандл не попадает.

## Что генерим

1. **Полированное описание карточки** — из сырого/пустого описания фида → маркетинговый текст.
2. **SEO** — `title` и `meta description` под товар.
3. **(опц.) Нормализация атрибутов** — маппинг кривых `<param>` фида к каноническим ключам (см. `DATA-MODEL.md`).

## Кеширование (обязательно)

Не пересчитываем неизменное: хэшируем вход (`title + description + ключевые attributes`) → если хэш совпал с прошлой сборкой, берём сохранённый текст. Так повторные сборки почти не тратят токены.

## Промпты (RU)

**Описание карточки:**
```
Ты — копирайтер мебельного магазина. Перепиши данные о товаре в продающее,
но честное описание на русском. 2–4 коротких абзаца, без воды и кликбейта,
упомяни материал, размеры и сценарий использования, если они есть в данных.
Не выдумывай характеристики, которых нет.

Товар: {title}
Сырое описание: {description}
Характеристики: {attributes}
```

**SEO:**
```
Сгенерируй SEO для карточки товара. Верни строго JSON без пояснений:
{"title": "...", "description": "..."}
title ≤ 60 символов, description ≤ 155 символов, на русском, с типом товара
и ключевой характеристикой, без спама и КАПСА.

Товар: {title}. Характеристики: {attributes}.
```

(JSON-ответы безопасно парсить с try/catch, сняв возможные ```-ограждения.)

## Модель и бюджет

- Подойдёт младшая модель (быстрая/дешёвая) — задача простая. Дорогие модели для карточек не нужны.
- Прикинь бюджет: ~`число_товаров × ~1k токенов`. Для каталога в сотни товаров — недорого; благодаря кешу платишь в основном за первую сборку и за новые/изменённые товары.

## Точки в пайплайне

`sync` → (`enrich` опц.) → `embed` → `build`. `enrich` заполняет `descriptionRich` и `seo` в `catalog.json`; если шаг пропущен, карточки используют сырое `description`.
